## Context
James declared DCD is pivoting from alert/prediction dashboard to pattern matching engine. "The brain doesn't predict. It MATCHES." Query interface = 3 dropdowns (What/Where/When) + search. Fusion = the differentiator. Core metaphor: "the fisherman picks where to fish." Previous UX was 25-panel workbench + 27-layer map. New UX must communicate "3.2M records, you search, we match + show what else was happening."

## Key Themes Identified
1. **Search bar IS the product** — landing page = query, not dashboard
2. **Live record counter** — as you set filters, show how many records match BEFORE you search
3. **The Coincidence Score** — co-occurrence frequency vs. chance baseline = the unique metric
4. **Fusion as sequential reveal** — peel layers, don't dump everything
5. **Brain as character** — daily diary, personality, humility about track record
6. **Time Machine query** — reconstruct any historical date across all domains
7. **Fingerprint overlay** — visual pattern recognition via stacked domain bar charts
8. **Embeddable widgets** — distribution strategy, brain lives everywhere
9. **Domain dot columns** — in search results, consistent color columns reveal patterns at a glance
10. **"Prove Me Wrong" mode** — adversarial query as engagement

## Ideas by Category

### Landing Page (5)
1. The Google Moment — search bar + 3 dropdowns, dark screen, trending queries
2. Recipe Wall — 8-12 pre-computed pattern cards the brain finds interesting TODAY
3. Split Second — one AI sentence, one number (3.2M), then search
4. The Bookshelf — library-catalog view of brain by domain, sparkline spines
5. Proof Shot — single best graded arc as the first thing you see

### Query Interface (5)
1. Three-Knob Console — WHAT/WHERE/WHEN with live match counter
2. Filter Funnel — progressive filtering with animated count-down from 3.2M
3. Combination Lock — rotating dials with green/yellow/red data density feedback
4. "More Like This" — re-query using any result's embedding vector
5. Recipe Builder — pin results to a board, cross-query for co-occurrences

### Pattern Match Visualization (5)
