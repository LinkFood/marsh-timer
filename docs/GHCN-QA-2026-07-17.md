# GHCN-Daily Archive QA — 2026-07-17

**Origin:** THE TOMATO QUESTION (docs/THE-WEEK.md, 07-17). One reader asking "when do I
plant tomatoes" exposed MD ghcn-daily rollups carrying `min_temp_f = 7.0` stuck for
weeks of summer 2002-2004 — a broken station that rode ingest with no plausibility
screen and fabricated a "June 30, 2004 freeze." This is the archive-wide answer:
survey all 50 states, flag confirmed artifacts (supersede-don't-delete), fix the
faucet, name the consumers.

**Scanner:** `scripts/ghcn-qa-scan.ts` (survey + flag pass, rerunnable, idempotent).
**Scope:** every ghcn-daily row in hunt_knowledge — 1,390,141 rows, 50 states,
1950-2025. **Full artifact dump (row ids, notes, suspects):** written by the scan to
`/tmp/ghcn-qa-full.json` (regenerate anytime; survey is read-only, ~10 min).

---

## 1. The verdict in one paragraph

**5,266 confirmed artifact rows across 44 of 50 states** (clean: AL, AR, DE, RI, SC,
SD), all now carrying `metadata.qa_flag` + a human-readable `qa_note` receipt. The MD
stuck-7 sensor was not an isolated case — it is one instance of a **national family of
stuck/sentinel instrument failures**, dominated by three signatures: Hawaii's
0°C-conversion sentinel (`min_temp_f = 32.0` pinned up to **237 consecutive days**,
2012-2025, 2,491 rows — HI's entire "state minimum" was a dead instrument for most of
a decade), the **-40 sentinel** (-40°C = -40°F; Alaska carried it 37 consecutive
August days at 86.5°F below the state average; Colorado summers carry it too, plus a
physically impossible `min = -148`), and the 2000-2007 stuck-sensor era across the
East (MD's 7.0, VA's 32.0, and cousins in NC, PA, OH, KY, IN, ME, OK, KS). Zero
`min > max` inversions exist archive-wide.

## 2. Artifact signatures scanned

| # | Signature | Confirmation rule (flag only when confirmed) |
|---|-----------|----------------------------------------------|
| a | **Stuck run** — identical `min_temp_f` (or `max_temp_f`) on ≥5 consecutive days | median spread vs state average > 30°F across the run; OR ≥10-day pin sitting > 15°F from average (real plateaus pin NEAR the average — HI max 89 for 10 days is weather, MD min 7 in December is not); OR the run contains a seasonal impossibility; OR the same state+field+value was confirmed broken in another run (stuck sensors stay stuck across seasons) |
| b | **Seasonal impossibility** — `min ≤ 15°F` May-Sep outside AK; `max ≥ 115°F` Dec-Feb | min: spread vs state avg low > 38°F (the verified guard-A broken-instrument ceiling; real Rockies cold sinks top out ~35). max: unconditional (US winter record high ≈ 100°F) |
| c | **Inversion** — `min_temp_f > max_temp_f` | unconditional (includes the old backfill's `Infinity → 0` no-reading sentinel) |

Thresholds are the planting wave's verified guards (scripts/frost-climatology.ts
guards A/B/D) generalized to the whole lane, so the faucet, the archive flags, and
the climatology agree on what "broken" means.

## 4. What was flagged (supersede-don't-delete)

Flag pass of record: **5,266/5,266 patched, 0 failed, 0 pre-existing** (run
`ghcn-qa-2026-07-17`, log /tmp/ghcn-qa-flagrun.log), spot-verified live on the MD
2004-06-30 "freeze" day, the AK August -40 window, and the HI 2018 sentinel. Every
confirmed row got three metadata keys via read-merge-PATCH (no row deleted, no other
metadata touched, content and embedding untouched):

```
qa_flag : min-max-inversion | implausible-min-stuck | implausible-max-stuck
          | implausible-min-seasonal | implausible-max-seasonal
qa_note : e.g. "min_temp_f=7 stuck 16d (2003-09-04..2003-09-19, median spread 52.3F)"
qa_run  : "ghcn-qa-2026-07-17"
```

Readers respect flags the same way they already respect storm-event supersedes:
`.is('metadata->qa_flag', null)` — or keep the row and null out `min_temp_f` /
`max_temp_f` at read when `qa_flag` is present (the avg_* fields on a flagged row are
station means and remain usable).

## 5. The faucet fix (`scripts/backfill-ghcn-daily.ts`)

The old ingest aggregated every station reading unscreened and wrote `0` when no
station reported a min/max (the `Infinity → 0` sentinel — fabricated "coldest reading
was 0°F" narratives). Fixed at the source, so re-ingests and future backfills cannot
recarry the artifact:

1. **Three-pass aggregation.** Pass 1 parses stations and computes preliminary
   per-day state averages (the spread baseline). Pass 2 screens each STATION's
   readings with the same rules as the archive scan — stuck-run confirmation
   (spread/length/seasonal/known-bad-value propagation), seasonal impossibility,
   station-day min>max inversion (drops both readings). Pass 3 aggregates only the
   survivors, so the state extreme recomputes from working instruments instead of
   being discarded for the day.
2. **Rejection counts logged per state-year** — `plausibility screen rejected N
   station readings (stuck min / stuck max / seasonal min / seasonal max /
   inversions)` — nothing is dropped silently.
3. **Sentinel killed:** days with no reported low/high now write `min_temp_f` /
   `max_temp_f` as `null`, and the narrative only speaks of extremes that exist.

Verified against the known-bad window: re-running `ONLY_STATES=MD YEAR_FROM=2004
YEAR_TO=2004` rejects **226 station readings** (199 stuck-min — the 7°F sensor ran
most of that year — 26 seasonal-min, 1 inversion) and inserts nothing (idempotency
diff), i.e. the screen fires and the write path stays quiet on already-present data.

No live edge function writes ghcn-daily rows — the backfill script (standalone or via
orchestrator-v2 pipe) is the lane's only faucet, so this one fix closes it.
